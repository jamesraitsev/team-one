# SOP Library System

## Summary

Create one reusable SOP library for agent and project workflows, with SDLC as one category rather than the entire system.

## Problem

Repeated operating procedures are currently implied across documents instead of being stored in one reusable library that agents can run consistently across repos and initiatives.

## Expected Outcome

- one top-level reusable SOP library
- clear category model
- explicit agent and human approval boundaries
- linkable SOP usage across issues, decisions, journals, and projects
