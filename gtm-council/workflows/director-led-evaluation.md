# Director-Led Evaluation Workflow

Use this workflow when the user asks GTM Council to evaluate an opportunity.

## Inputs

Start with `templates/evaluation-brief.md`. If information is missing, proceed with labeled unknowns unless the missing information makes evaluation impossible.

## Steps

1. Clarify the evaluation objective.
2. Identify known facts, assumptions, and unknowns.
3. Choose participating agents using `frameworks/agent-activation.md`.
4. Assign distinct scopes to each specialist.
5. Request independent reviews using `templates/specialist-response.md`.
6. Map agreement, disagreement, and missing evidence.
7. Build the assumption register and select the top three decision-controlling assumptions.
8. Run debate using `frameworks/debate-protocol.md`.
9. Record the disagreement ledger and validation roadmap.
10. Collect independent votes using `templates/vote-card.md`.
11. Synthesize using `templates/final-report.md`.
12. State the final verdict and confidence.

## Director Guardrails

- Do not let specialists converge prematurely.
- Do not average votes into a false consensus.
- Do not suppress minority views.
- Do not fill missing evidence with confident narrative.
- Do not activate extra agents when the question is narrow.

## Output Requirements

The final report must include:

- the standard evaluation structure
- evidence labels for material claims
- independent vote results
- activation rationale
- assumption register
- disagreement ledger
- unresolved disagreements
- critical assumptions
- validation roadmap with confirming and disconfirming signals
