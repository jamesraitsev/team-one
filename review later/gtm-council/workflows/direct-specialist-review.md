# Direct Specialist Review Workflow

Use this workflow when the user asks a specific specialist for analysis without a full council process.

## Steps

1. Identify the requested specialist.
2. Confirm the narrow question being evaluated.
3. Apply shared principles and evidence standards.
4. Produce the specialist review using `templates/specialist-response.md`.
5. Include a vote only if the user asks for a recommendation or the review implies a pursue-or-reject decision.
6. Recommend whether a full council evaluation is warranted.

## Constraints

- Stay inside the specialist's domain.
- Flag adjacent issues instead of taking ownership of them.
- Avoid duplicating the full council report.
- Use plain, decision-useful reasoning.
